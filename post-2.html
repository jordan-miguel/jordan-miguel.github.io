I want to briefly go over the difference between HTTP and HTTPS for those who are interested.

What is HTTP?

Firstly, the acronym stands for Hypertext Transfer Protocol. Essentially HTTP allows an internet application to establish and improve the client-server communication between shared information systems.

In layman terms, your browser sends a HTTP request to the web server (a could computer for example). The server receives this request and if all is well, executes an application. It then returns the execution to the client.

If you click on a Youtube video, you are sending a request to the Youtube server, this server executes your request and plays the video - you are the client.

Simple, right?

What is HTTPs?

This stands for Hypertext Transfer Protocol Secure. You're probably thinking, secure is better - and you are completely right.
A HTTPs request follows the same flow as we've just described above in HTTP. There is one major difference.

Unlike a HTTP request, HTTPS requests secure the flow of information between client and server via encryption. Therefore this information cannot be intercepted and used in a malicious way.

Up until a few years ago HTTP was widely used and accepted as the norm, however web browsers place the integrity and confidentiality of data between client and server at the forefront of their concerns.

Checking your code

If you're working solely, or in a large team with a huge codebase you can check that you're conforming to HTTPs protocol best practices by using this quick tip.
Hit CMD+F or CTRL+F and search for any instances of 'http:/' - simply replace all with 'https:/' or one by one depending on your needs.

Note: If you are working in a large team that uses version control I would advise against doing this in the Master branch!
